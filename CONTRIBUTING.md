# Contributing

Thanks for contributing to Motorsport Electronics.

This repository is intended to make motorsport electronics files easier to share, improve, and reuse across the community.

## Licence

Unless otherwise stated, content in this repository is published under **CC0 1.0 Universal**.

By submitting files, screenshots, or other content to this repository, you confirm that:

- you created the content yourself, or you have the right to share it
- you are happy for it to be published under **CC0 1.0 Universal**
- you understand that others may copy, modify, redistribute, and reuse it without attribution

## How to contribute

There are two ways to submit a file:

### Option 1: Pull request

Use this if you are comfortable with GitHub.

1. Fork the repository
2. Add your files in the appropriate folder
3. Commit your changes
4. Open a pull request with the requested details

### Option 2: Open an issue

Use this if you just want to submit a file without dealing with Git.

1. Open a new **File submission** issue
2. Fill in the template
3. Attach the file and screenshots where relevant
4. A maintainer will review it and add it to the library if suitable

## What to include

Please include the following wherever possible:

- file name and short title
- author name or handle, if you want it shown
- manufacturer or platform
- target hardware or software
- firmware or software version
- short description of what the file does
- screenshot where relevant
- setup notes
- notes on fonts, icons, images, CAN databases, or other external assets used
- any dependencies or special instructions

## Folder structure

Please place files in the most appropriate location for the hardware, software, or file type.

Example structure:

- `CAN/`
- `Dash/`
- `ECU/`
- `PDM_PMU/`
- `Displays/`
- `Docs/`
- `Other/`

Within those folders, use clear subfolders where helpful, for example:

- manufacturer
- hardware model
- car or engine platform
- road / race / rally / drift / endurance
- verified / unverified

## File naming

Use clear names that help others understand what the file is.

A good format is:

`platform_or_manufacturer_file-description_version_author`

Examples:

- `ecumaster_adu7_rover-v8-track_v1_mike`
- `link_can-lambda-template_v2_jbloggs`
- `haltech_ecu-base-map_2.0_user123`

Avoid names like:

- `final`
- `new one`
- `config2`
- `test file`
- `latest latest`

## Submission rules

Do not submit:

- files you did not create or do not have permission to share
- paid or commercial content you are not allowed to redistribute
- copyrighted logos, graphics, fonts, or backgrounds without permission
- confidential customer, team, or company files
- malicious, misleading, or unsafe files

## Maintainer review

Maintainers may:

- rename files for consistency
- move files into a better folder
- request missing information
- reject submissions that cannot be legally or safely shared
- remove files later if ownership or rights are disputed

## Verification

A file being present in this repository does not guarantee that it is safe, complete, tested, or suitable for your setup.

Users are responsible for checking files before use.

## Questions

If you are unsure how to submit, open an issue and ask.
